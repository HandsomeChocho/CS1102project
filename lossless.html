Stage 1: Preprocessing

Purpose: Increase data compressibility by utilizing contextual data.

Techniques:

  • Run-length Encoding: Simplifies sequences of repeated patterns into pairs indicating the count and the pattern itself (e.g., Input: "0000000011111000001111", RLE Output: (8, '0') (5, '1') (4, '0') (4, '1')).
  • Frame Differencing (for video streams): Reduces redundancy by using the changes between frames, which frequently results in a large number of zeros that may be effectively compressed using run-length encoding.

Stage 2: Probability Model

Purpose: Acts as the central component of lossless compression by specifying the encoding of input data patterns according to their frequency of prevalence.

Process:

  1. Patterns occurring more frequently (high probability) are encoded with shorter codes.
  2. Less frequent patterns (low probability) receive longer codes.

Outcome: If the input data aligns well with the probability model, more frequent use of shorter output patterns occurs, which leads to a smaller compressed output size.

Stage 3: Probability Coding

Purpose: Applying the defined probability model-based coding to the incoming data.

Techniques:

  • Huffman Coding: Optimal when all probabilities are powers of two, generating the shortest possible codes under such conditions.
  • Arithmetic Coding: The whole message is encoded using intervals on the number line that correspond to the probabilities of symbol sequences, into a single number.
  • Lempel-Ziv-Welch (LZW): Using a dictionary-based technique, sequences can be efficiently encoded as single output values.
  • Lempel-Ziv 4 (LZ4): Offers a high speed for both compression and decompression, suitable for performance-critical applications.
  • Zstandard (ZSTD): Developed by Facebook, this real-time compression algorithm provides high compression ratios and speed, widely supported by numerous platforms.
